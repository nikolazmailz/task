6. Базу данных использовал hsqldb in-memory - поэтому дампа базы нету, при каждом новом старте проекта
 база заполнится данными из \main\resources\test-data.sql
7. Прошу прощения за оставленные TODO в коде
8. Так же не успел реализовать функционал:
    - bat скрипт запуска приложения.
    - Задание выполняемой команды (функции), а также имени файла рекомендуем сделать через параметры командной строки. Пример: test.bat sync test.xml
    - не проработал все возможные exceptions и проверки
    - Пользователь должен иметь возможность указывать имя файла по содержимому которого будет осуществляться синхронизация данных в таблице.
    - Пользователь должен иметь возможность указывать имя файла, в который будет осуществлена выгрузка.
9. Файл лога должен писаться в C:\\tmp\\log4j-application.log если диска С у вас нет то в \main\resources\log4j.properties в log4j.appender.file.File нужно прописать иной
   писать просто в диск C:\\log4j-application.log нельзя будет ошибка доступа
